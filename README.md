# T&G Mockup Builder

نظام تكوين الطلبات الموحد (Uniform Mockup Builder) مخصص لشركة T&G.

## طريقة الرفع على GitHub لتشغيل الموقع

لقد قمت بتجهيز الكود ليعمل مباشرة على **GitHub Pages** بدون أي مشاكل (تم حل مشكلة الصفحة البيضاء عن طريق ضبط المسارات النسبية).

### الخيار الأول: الرفع اليدوي (سهل وسريع)
1. قم بإنشاء مستودع جديد (Repository) على GitHub.
2. قم برفع جميع الملفات الموجودة هنا إلى المستودع.
3. انتظر قليلاً، لقد أضفت ملف "GitHub Actions" في مجلد `.github/workflows` سيقوم ببناء الموقع ورفعه تلقائياً.
4. اذهب إلى **Settings** ثم **Pages** في مستودعك على GitHub، وتأكد من اختيار **GitHub Actions** كمصدر (Source).

### الخيار الثاني: استخدام نسخة الـ HTML المنفردة
إذا كنت تريد ملفاً واحداً فقط يعمل في أي مكان بدون الحاجة لعملية بناء (build):
- استخدم الملف `tg-uniform-standalone.html`.
- يمكنك تغيير اسمه إلى `index.html` ورفعه وحده وسيعمل فوراً.

## المميزات المضافة للتوافق مع GitHub:
- تم ضبط `base: './'` في ملف `vite.config.ts` لضمان تحميل الملفات البرمجية بشكل صحيح.
- تم إضافة ملف Automated Deployment (GitHub Actions).
- تم التأكد من أن جميع المسارات تعمل بشكل نسبي.
